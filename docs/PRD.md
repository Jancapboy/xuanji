# PRD — 玄机 (XuanJi) · AI 命理分析系统

> **版本**: v0.1.0-MVP  
> **定位**: 传统命理学 × 现代 AI 的融合分析平台  
> **核心差异**: 以结构化知识库为引擎，AI 为解释器，不是"瞎编"而是"有典可依"

---

## 1. 产品概述

### 1.1 一句话描述
一个基于知识图谱驱动、AI 增强的传统命理分析工具，让命理咨询从"算命先生拍脑袋"变成"典籍+数据+AI推理"的科学化解读。

### 1.2 目标用户
- 命理爱好者（想自学但典籍难懂）
- 职业命理师（需要知识库辅助和案例参考）
- 普通用户（好奇自己的命盘，想要有深度的解读）

### 1.3 核心价值主张
> **"不是 AI 在算命，是 AI 在帮你读典籍、解命盘"**

## 2. 功能模块

### 2.1 命盘系统（核心）

#### 2.1.1 八字排盘
- **输入**: 阳历/阴历出生日期时间、出生地（用于真太阳时校正）
- **输出**: 
  - 四柱（年柱、月柱、日柱、时柱）
  - 天干地支、藏干、十神
  - 五行力量分布图
  - 纳音五行
- **交互**: 可视化命盘，点击任一柱查看详细解读

#### 2.1.2 紫微斗数排盘
- **输入**: 同上
- **输出**:
  - 十二宫位图
  - 主星、辅星、煞星分布
  - 四化（禄、权、科、忌）飞布
  - 大限、流年盘
- **交互**: 交互式星盘，宫位置换查看

#### 2.1.3 塔罗抽牌
- **模式**: 单张牌、三张牌（过去-现在-未来）、凯尔特十字
- **交互**: 洗牌动画 → 抽牌 → 翻转揭示 → AI 结合牌义解读

### 2.2 知识库系统（核心引擎）

> **这是整个产品的心脏。所有 AI 解读都基于知识库，不是幻觉生成。**

#### 2.2.1 知识图谱结构

```
知识层（四层架构）
│
├─ 经典文献层
│  ├─ 《子平真诠》
│  ├─ 《紫微斗数全书》
│  ├─ 《三命通会》
│  └─ ...（可扩展）
│
├─ 规则引擎层
│  ├─ 五行生克: 金→木, 水→火, ...
│  ├─ 十神关系: 正官、偏官、正印、偏印...
│  ├─ 宫位含义: 命宫、财帛、夫妻...
│  ├─ 星曜性质: 紫微主贵、贪狼主欲...
│  └─ 格局判断: 君臣庆会、府相朝垣...
│
├─ 案例库层
│  ├─ 历史名人命盘（公开资料）
│  ├─ 典型格局案例
│  └─ 用户授权匿名案例
│
└─ 现代解读层
   ├─ 心理学映射（MBTI、大五人格）
   ├─ 统计学验证（大数据趋势）
   └─ 社会学视角（时代背景影响）
```

#### 2.2.2 知识检索（RAG）
- **向量化**: 知识条目 → 文本嵌入 → 向量索引
- **混合检索**: 向量相似度 + 关键词匹配 + 图谱关系遍历
- **上下文组装**: 检索到的知识片段 + 命盘数据 → 送入 AI 提示

#### 2.2.3 知识库管理
- **浏览**: 按分类浏览知识条目
- **搜索**: 全文检索
- **贡献**: 用户可提交知识（审核后入库）
- **版本**: 知识条目可追踪修改历史

### 2.3 AI 解读引擎

#### 2.3.1 解读流程
```
用户命盘 → 知识库检索相关规则/案例 → 组装上下文 → AI 生成解读 → 标注引用来源
```

#### 2.3.2 解读类型
- **性格分析**: 基于日主、命宫主星
- **事业财运**: 财官印分析、事业宫星曜
- **感情婚姻**: 夫妻宫、桃花星、合婚分析
- **健康提示**: 五行偏枯、疾厄宫
- **流年运势**: 大运流年交互分析
- **合婚配对**: 双方命盘交互分析

#### 2.3.3 可解释性
- 每条解读必须**标注知识来源**（出自哪本典籍、哪条规则）
- 提供"为什么这样解读"的推理链
- 区分"典籍记载"和"AI 推测"

### 2.4 对话系统

- **咨询模式**: 用户提问 → 检索知识库 → AI 基于命盘+知识回答
- **会话历史**: 保存对话记录，支持回溯
- **上下文感知**: AI 记住当前命盘和对话历史

### 2.5 运势追踪

- **大运流年**: 自动计算当前所处大运和流年
- **每日运势**: 基于日柱和当日干支的简要提示
- **运势日历**: 可视化展示吉凶日

## 3. 技术架构

### 3.1 技术栈
| 层级 | 技术 |
|------|------|
| 前端 | React 18 + Vite + TypeScript + Tailwind CSS |
| 状态 | Zustand |
| 存储 | IndexedDB（本地）+ 可选后端同步 |
| AI | Kimi API（kimi-for-coding / kimi-k2.7-code） |
| 知识库 | 本地 JSON + 简单向量索引（初期） |
| 可视化 | D3.js / ECharts（命盘图表） |
| 构建 | GitHub Actions → GitHub Pages |

### 3.2 知识库实现（MVP 简化版）

```typescript
// 知识条目结构
interface KnowledgeEntry {
  id: string;
  category: 'classic' | 'rule' | 'case' | 'modern';
  source: string;        // 来源典籍/文献
  title: string;
  content: string;
  tags: string[];        // 标签，用于检索
  relatedConcepts: string[]; // 关联概念
  confidence: 'certain' | 'probable' | 'speculative'; // 可信度
  metadata: {
    author?: string;
    dynasty?: string;
    page?: string;
  };
}

// 知识图谱索引（简化）
interface KnowledgeGraph {
  entries: Map<string, KnowledgeEntry>;
  conceptIndex: Map<string, Set<string>>; // 概念 -> 条目ID集合
  tagIndex: Map<string, Set<string>>;     // 标签 -> 条目ID集合
  // MVP 阶段不用真向量，用关键词倒排索引
}
```

### 3.3 数据流

```
用户输入命盘信息
    ↓
排盘引擎计算四柱/紫微星盘
    ↓
知识检索器：提取命盘关键特征 → 检索相关知识条目
    ↓
上下文组装器：命盘数据 + 知识条目 + 用户问题 → Prompt
    ↓
AI 生成解读（带引用标注）
    ↓
前端渲染：命盘可视化 + 解读文本 + 知识来源卡片
```

## 4. UI/UX 设计

### 4.1 视觉风格
- **主题**: 东方古典 × 现代极简
- **色彩**: 
  - 主色：墨黑 `#1a1a2e`、朱砂 `#c23a30`
  - 辅色：宣纸白 `#f5f0e8`、青金石 `#2e5090`
  - 五行色：金白、木青、水黑、火赤、土黄
- **字体**: 标题用衬线体（模拟古籍），正文用无衬线
- **动效**: 抽牌动画、星盘旋转、墨迹渲染效果

### 4.2 核心页面

#### 4.2.1 首页 / 命盘输入
- 出生信息表单（日期、时间、地点）
- 快速排盘按钮
- 历史命盘列表

#### 4.2.2 命盘展示页
- 八字：四柱排布 + 五行力量条形图
- 紫微：十二宫格图 + 星曜标记
- 右侧：AI 解读面板（可切换不同主题）
- 底部：知识来源引用列表

#### 4.2.3 知识库页
- 分类导航（经典/规则/案例/现代）
- 搜索栏
- 知识卡片列表
- 条目详情页（显示引用次数、相关命盘特征）

#### 4.2.4 对话页
- 侧边栏：命盘摘要
- 主区域：对话气泡
- 输入框：支持快捷问题按钮

### 4.3 交互细节
- **命盘点击**: 点击任意柱/宫 → 弹出该位置的知识解读
- **解读溯源**: 点击解读中的引用标记 → 跳转到知识库对应条目
- **对比模式**: 可并排显示两个命盘（用于合婚）

## 7. 非功能性需求

- **隐私**: 所有命盘数据本地存储，不上传（除非用户主动同步）
- **可离线**: MVP 排盘和知识库查询可完全离线
- **AI 调用**: 仅在需要 AI 解读时联网调用 API
- **响应速度**: 排盘 < 100ms，知识检索 < 500ms，AI 解读 < 5s
